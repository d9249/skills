  - full name: `p0deje/Maccy`
  - description: `Lightweight clipboard manager for macOS`
  - homepage: `https://maccy.app`
  - language: Swift
  - license: MIT

- README:
  - Maccy works on macOS Sonoma 14 or higher.
  - Official install command is `brew install maccy` — not `brew install --cask maccy`, even though Homebrew exposes cask metadata internally.
  - Main shortcut: `Shift+Command+C` to open Maccy.
  - Enter copies selected item, `Option+Enter` pastes, `Option+Shift+Enter` pastes without formatting.
  - `Option+P` pins, `Option+Delete` deletes.
  - Sensitive-copy controls: `ignoreEvents`, menu icon with Option, Option+Shift for next-copy ignore.
  - Default ignored pasteboard types include transient/concealed/autogenerated types and several known password-manager/app types.
- GitHub release / Homebrew / Sparkle:

  - asset: `Maccy.app.zip`
  - Homebrew cask version: `2.6.1`, `depends_on macos >= 14`, `auto_updates true`

  - Settings panes show General, Storage, Appearance, Ignore, Advanced; useful to summarize as practical setup areas rather than source-code detail.

Downloaded official assets and extracted useful demo frames from the MP4. Saved:

Pattern: when a tool's README lacks screenshots but the official website has a demo video, use `ffmpeg` to extract sharp representative frames. Avoid generic GitHub OG images if the extracted official frame shows the product surface better.

Example:

```bash
ffmpeg -y -ss 5 -i /tmp/maccy-assets/Demo.mp4 -frames:v 1 /tmp/maccy-assets/maccy-demo-frame-5s.png
```

## Writing choices

- Platform slug: `macos-linux` only, because the tips site has no dedicated macOS slug. Explicitly state in body/highlights that Maccy is macOS-only and requires macOS Sonoma 14+.
- Status: `Open source macOS utility`.
- Do not imply Linux support despite the site's `macOS / Linux` label.
- Installation section should preserve the README's exact command: `brew install maccy`.
- Body should balance convenience with clipboard-history risk:
  - mention Accessibility permission for automatic paste
  - mention password-field shortcut caveat from README FAQ
  - recommend checking Storage and Ignore settings before using heavily
  - list default confidential/temporary pasteboard types, but avoid claiming all secrets are always protected

## Pitfalls surfaced

1. Homebrew can represent GUI apps as casks, but the official README command may still be `brew install maccy`. Quote the official command and separately use cask metadata for version/minimum macOS evidence.
2. Clipboard managers need a stronger privacy caveat than ordinary menu-bar utilities. Call out token/password/customer-data risk, Ignore rules, clear-on-quit, and system clipboard clearing.
3. Official website assets can be more useful than README assets; inspect linked homepage before falling back to GitHub Open Graph.
4. macOS-only apps still use `macos-linux` in this site today. Always state real OS support in text and highlights.
